Configuration Guide

Learn how to configure our platform for your specific needs.

Configuration Files

Our platform uses several configuration files to control its behavior. The main configuration file is config.yaml, which is located in the following directory:

  • Windows: C:\Users\<username>\AppData\Local\OurPlatform\config.yaml
  • macOS: /Users/<username>/Library/Application Support/OurPlatform/config.yaml
  • Linux: /home/<username>/.config/ourplatform/config.yaml

Note

If the configuration file doesn't exist, you can create it manually or run the platform once to generate the default configuration.

Basic Configuration

Here's a basic example of a configuration file:

YAML 
# Basic configuration
api:
  port: 3000
  host: localhost
  timeout: 30000  # milliseconds

database:
  type: postgres
  host: localhost
  port: 5432
  username: user
  password: password
  database: myapp

logging:
  level: info
  file: logs/app.log
  max_size: 10  # MB
  max_files: 5

security:
  enable_ssl: true
  ssl_cert: /path/to/cert.pem
  ssl_key: /path/to/key.pem

Environment Variables

You can also configure the platform using environment variables. Environment variables take precedence over configuration file settings.

Environment variables should be prefixed with PLATFORM_ followed by the configuration path with underscores. For example:

Shell 
# Set database configuration via environment variables
export PLATFORM_DATABASE_HOST=db.example.com
export PLATFORM_DATABASE_PORT=5432
export PLATFORM_DATABASE_USERNAME=production_user
export PLATFORM_DATABASE_PASSWORD=secure_password

Security Warning

Be careful with sensitive information like passwords in environment variables, as they might be visible in process listings or logs.

Advanced Configuration

Profiles

You can create different configuration profiles for different environments (development, testing, production):

YAML 
default: &default
  api:
    port: 3000
    host: localhost
    
development:
  <<: *default
  logging:
    level: debug
    
production:
  <<: *default
  api:
    port: 80
    host: 0.0.0.0
  logging:
    level: warn

To use a specific profile, set the PLATFORM_PROFILE environment variable:

Shell 
export PLATFORM_PROFILE=production
